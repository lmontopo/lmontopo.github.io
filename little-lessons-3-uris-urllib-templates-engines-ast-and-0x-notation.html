What is a URI? Is it the same as a URL?

URI stands for uniform reserouce identifier. A very common form of a URI is a URL, uniform resource locator. The URL is just the address of a website - ie. the thing you type into the browser when you want to go somewhere. And while we're talking about URL's...

What meant by the 'path' and the 'query' of a URL?

The wikipedia page, here, does a pretty good job at explaining this, but I'll summarize it here also. The URL consists of several different parts. Lets discuss these parts with respect to this example: http://www.cineplex.com/Search?Query=Interstellar

  • Protocol: specifies which protocol is being used to obtain the resource. In our example it is 'http'
  • Domain Name: This can be an IP address, or a registered, easier to remember, name that is associated to an * IP address. The example's domain name is 'cineplex.com'
  • The Port: After the domain name, one can optionally specify a port number. By default we are directed to port 8000, and so it is usually unnecessary to manually specify a port number.
  • Path: Within a single domain name or IP address there can be several different files. The path specifies the address of the particular file we are requesting. In the example this is specified by /Search
  • The Query String: contains data, usually inputed by the user, which will be passed to some sort of software running on the servers computer. In my example, I typed in Interstellar on the search bar of the cineplex website. "Interstellar" was passed to the servers computer, used in some program to figure out the optimal output, and then the resulting info, in the form of a webpage, was passed back to me. The Query string becomes part of the URL, "?Query=Interstellar". The Query string always (as far as I know) begins with a question mark.

Lets inspect the following URL: http://www.cineplex.com/Search?Query=Interstellar
